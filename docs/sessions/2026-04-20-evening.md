## Gotchas captured for future-me

- **`.dropDestination(for: URL.self)` with `isTargeted:` as a trailing closure** looks weird because you're passing TWO closures to the modifier (action + target-state callback). The `isTargeted:` version takes a `Bool -> Void`, not a `Binding<Bool>` — common mistake is to write `isTargeted: $isDropTargeted` which is the older `.onDrop` signature.
- **`TimelineView(.periodic(from:by:))` starts firing from the `from:` instant** — if you pass `.now` at view-construction time the first tick is immediate, which is usually what you want. Passing `scanStartedAt` directly (as a `Date`) is accepted because `Date` conforms to `Comparable` with the `TimelineSchedule` requirement.
- **`@Observable` + `private(set)`**: SwiftUI still observes reads through the computed property access, so `private(set)` doesn't break observation — it just prevents external writes, which is exactly what we want for a state machine where only internal methods (`start`, `cancel`, `applyBatch`) may transition.
- **Feedback-loop discipline: output styles are defaults, not laws.** The `learning` output-style asked for "5–10 lines of meaningful code from the user." Applied mechanically to Wave 4.1's `consume` loop, that meant handing the user a `fatalError` stub for code with no remaining design choice. User pushback triggered an inline feedback memory so next session doesn't repeat it. Broader lesson: output-style modes configure the DEFAULT behaviour; individual turns should still apply judgment about whether the current task actually matches the pattern the style is optimised for.
