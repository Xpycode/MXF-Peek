# Session: 2026-04-21 (continued) — P5 bundle-script cleanup

## Goal
P5 from Current Sprint — retrofit the two inherited P2toMXF shell scripts to match the post-ffprobe-pivot reality: one static binary, no dylib graph, and this project's `01_Project/AvidMXFPeek/Resources` path (not the fork's root-level path).

## Actions

**Created `bundle-ffprobe.sh`** (replaces `bundle-ffmpeg.sh`):
- Path: `01_Project/AvidMXFPeek/Resources` (was `AvidMXFPeek/AvidMXFPeek/Resources` — resolved to nothing in this layout).
- Dropped the Homebrew-location fallback chain. Homebrew ffprobe is dynamically linked against ~30 dylibs and is unusable under hardened runtime. Script now requires an explicit path arg.
- Added `otool -L | grep -vE '^(/usr/lib/|/System/)'` preflight. Any non-system dep aborts with a pointer to martin-riedl.de. Verified against the bundled binary (62 MB, system-framework-only) — passes cleanly.
- Added same-file copy guard using `cd + pwd` canonicalization (BSD-portable — no `realpath` on macOS). Prevents `cp: source and destination are identical` from short-circuiting `set -e` when someone re-runs with the bundled path.
- 30 → 50 lines, but logic density up, ceremony down.

**Rewrote `sign-bundled-binaries.sh`**:
- Path fix: same as above.
- Stripped the dylib-loop (`for dylib in lib/*.dylib`) — no dylibs post-pivot.
- Stripped the `bmxtranswrap / mxf2raw / ffmpeg` signing block — all three deleted in Wave 4.6-A.
- Kept: Apple Development cert SHA-1 default, temp entitlements file (`allow-unsigned-executable-memory` + `disable-library-validation`), `--options runtime --timestamp`, codesign -dv verification output.
- Early-exit if `$FFPROBE` doesn't exist — points at `./bundle-ffprobe.sh` as the prerequisite.
- 80 → 55 lines.

**Deleted `bundle-ffmpeg.sh`**.

## Verification
End-to-end smoke test: `./bundle-ffprobe.sh 01_Project/AvidMXFPeek/Resources/ffprobe && ./sign-bundled-binaries.sh`
- Bundle: version check reads `ffprobe 8.1-https://www.martin-riedl.de`, dylib check passes (no non-system deps), same-file guard activates → no-op copy.
- Sign: replaces existing signature, outputs `flags=0x10000(runtime) hashes=3779+7 location=embedded team=FDMSRXXN73`.
- No warnings, no errors.

## Decisions
No new architectural decisions. P5 is pure cleanup — the design was already settled by decision `2026-04-20-C` (ffprobe pivot) and the `/reorg` numbered-folder layout. Two small implementation choices worth flagging for future self:
- **Dylib preflight as hard-fail, not warn.** A Homebrew ffprobe slipping through would notarize fine, then crash at first launch on a user's machine. Fail-fast is the right posture.
- **Same-file guard via `cd + pwd` rather than `realpath`.** Cookbook pattern 38 ("destructive copy guard") applies, but `realpath` isn't preinstalled on macOS. BSD-portable idiom documented inline.
